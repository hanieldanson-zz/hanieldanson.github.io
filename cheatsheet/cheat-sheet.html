Variables are containers for data and can be either numbers or words (a string). The syntax follows: 

var *nameOfVariable* = *value*
var x = 5;
var message = “Hello world!”;
var empty; // undefined variable "empty."
// Undefined is not a string or integer.
// Note: you cannot create two variables with the same name

JavaScript Objects are similar to real objects. They have properties and you can do things to the them. In JavaScript, doing something to an object can be described as using a method on an object. Create a JavaScript Object with: 

Object Literal: Most common way to create object
var myObject = {}     // An empty object
var person = {      // An object with 2 items
  name: “Bob”;
  age: 19;
}
Object Constructor: Next most common way to create object
var person = new Object ();
person.name = “Bob”;
person.age = 19;

The Constructor pattern is another way to use the object 
constructor with this, referring to the global object.
function Person(theName, theAge) {
  this.name = theName;
  this.age = theAge;
}
var bob = Person(“Bob”, 19);

The Prototype pattern can also be used as an alternative 
for creating objects. For example:
function Person() { }
Person.prototype.name = “Bob”;
Person.prototype.age = 19;
var bob = new Person(); // calls constructor to create 
bob w/ the properties name = “Bob” and age = 19

Properties are values associated with a JavaScript object. Properties can be accessed two ways: 

Dot notation: obj.propertyName 
Square bracket notation: obj[propertyName]

Accessing and Enumerating Properties on Objects using a for/in loop: 

var school = {name:”DBC”,loc:”Chicago”,cohort1:”Squirrels”}
for (var eachItem in school) {
console.log(eachItem); // Prints name, loc, cohort1
}

Dot notation is easier to write and read. Square bracket notation allows access to properties containing special characters and selection of properties using variables. 

var school = {name:”DBC”,loc:”Chicago”,cohort1:”Squirrels”}
console.log(school.name); // DBC
console.log(school[“cohort” + 1]); // Squirrels

hasOwnProperty returns a boolean value indicating if the object being called has a property with the name of the argument in parenthesis. This is useful when enumerating properties of an object in the for...in construct. 

var school - {schoolName:”DBC”};
console.log(school.hasOwnProperty(“schoolName”));

Each data property has 3 attributes:

Configurable: Specifies if it can be deleted or changed.
Enumerable: Specifies if it can be returned in for/in loop.
Writable: Specifies if it can be changed.

Every JavaScript function has a prototype property (empty by default), and you attach properties and methods on this prototype property when implementing inheritance. The prototype property is not enumerable (inaccessible in for/in loop).

An object’s prototype attribute points to the object’s “parent”—the object it inherited its properties from. The prototype attribute is normally referred to as the prototype object, and it is set automatically when you create a new object.
