Breaking Down Enumerables

A brief look at the map method

11/23/2014

This week's technical blog post is about everyone's favorite enumerable method, map. Enumerable map is an extremely useful method that gives Rubyists' the flexibility to move outside of the typical each method constraints. One of the advantages of map is the flexibility it gives you with arrays. If you use each on an array and try to take the square root of the argument, it will simply spit out the same unaltered array. In this case, the following code isn't very helpful: 

arr = [4,16,64,256]
arr.each{ |x| x = Math.sqrt(x).to_i }
puts arr.join(", ")   #=>   4, 16, 64, 256
# Displays 1,2,3,4,5

In case you're wondering, making arr.each! destructive confuses Ruby even more. Don't try it unless you like seeing error messages. What is a poor Rubyist to do when each can't get the job done? Map, my friend, map is the answer. The cool thing about map is that it gives you the flexibility to either create a new modified array (non-destructive) or to modify the existing array. If we try that same code with map instead of each, we get a result that actually makes sense: 

arr = [4,16,64,256]
arr = arr.map!{|x| Math.sqrt(x).to_i }
puts arr.join(", ")   #=>   4, 16, 64, 256 
#=> 2, 4, 8, 16

Of course, map isn't constrained to the world of curly braces. You can also define a method before map, and then use that method to modify each element in the array as you iterate through it. I'm still getting the hang of symbols, but the example below includes a symbol combined with ampersand to reference the previously defined method. At first, you may be tempted to put parenthesis around the "&:x2". The code below stays dry and functions correctly without them: 

class Numb
  def x2
    self * 2
  end
end
[2,5,10].map &:x2
